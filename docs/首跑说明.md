# 首跑说明

这份说明面向第一次把 `meta-model-train` 拉到服务器后，如何快速确认：

- 代码已同步成功
- conda 环境可用
- GPU 训练脚本能正常运行

## 0. 进入项目并激活环境

```bash
cd /root/meta-model-train
source /root/miniconda3/etc/profile.d/conda.sh
conda activate mpp-meta
```

如果你的仓库目录名不是 `meta-model-train`，把上面的路径换成实际目录名即可。

## 1. 确认环境状态

```bash
git log --oneline -1
git status
python --version
python scripts/check_environment.py
```

期待看到：

- `git status` 基本干净
- Python 是 `3.11.x`
- `cuda_available` 为 `True`
- GPU 名称能显示 `NVIDIA A40`

## 2. 生成一个小的 architecture 样本

```bash
python scripts/enumerate_architectures.py --sample_size 8
```

成功后，应该会生成：

```text
artifacts/sampled_architectures_L8_n8_seed25.csv
```

## 3. 跑第一轮 smoke training

```bash
python scripts/train_minimal_arch_smoke.py --device cuda --max_steps 50
```

成功后，应该会看到持续打印的 `train_loss` / `val_loss`，并在下面目录生成结果：

```text
outputs/toy_diffusion/meta_model_smoke/<run_name>/
```

其中至少会有：

- `config.json`
- `metrics.csv`

## 4. 跑一个极小 family 训练

```bash
python scripts/train_minimal_arch_family.py \
  --architecture_csv artifacts/sampled_architectures_L8_n8_seed25.csv \
  --max_architectures 2 \
  --device cuda \
  --max_steps 30 \
  --val_every 10
```

这一步的目标不是正式实验，而是确认：

- 多个 architecture 可以顺序训练
- `family_summary.csv` 和 `family_manifest.json` 能正确生成

## 5. 进一步验证 meta dataset 聚合

把 `<family_name>` 换成刚才输出的实际目录名：

```bash
python scripts/build_meta_dataset.py \
  --family_dir outputs/toy_diffusion/meta_model_family/<family_name>
```

成功后，应生成：

```text
outputs/toy_diffusion/meta_model_family/<family_name>/meta_dataset.csv
```

## 一键最小链路

如果你只是想快速确认“脚本入口、训练、聚合”整条链都还活着，可以直接运行：

```bash
python scripts/run_smoke_pipeline.py --device cpu
```

或者统一从 CLI 入口运行：

```bash
python scripts/meta_model_cli.py smoke-pipeline --device cpu
```

服务器上如果想直接测 GPU，可改成：

```bash
python scripts/run_smoke_pipeline.py --device cuda
```

它会自动完成：

- 一个最小 smoke training
- 一个最小 architecture 枚举
- 一个最小 family training
- 一个 meta dataset 聚合

## 常见问题

### 1. `ModuleNotFoundError`

优先确认你是在仓库根目录下运行命令：

```bash
cd /root/meta-model-train
```

### 2. `torch.cuda.is_available()` 为 `False`

先检查：

```bash
nvidia-smi
```

如果这里都看不到 GPU，就不是 Python 脚本本身的问题。

### 3. 输出目录很多

这是正常的。训练输出默认都在 `outputs/`，它已经被 `.gitignore` 忽略，不会进入 Git。
