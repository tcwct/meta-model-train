# 服务器正式训练说明

这份说明面向当前 `meta-model-train` 项目的下一阶段目标：

- 优先提高单位墙钟时间内产出的曲线数据量
- 使用服务器做分片训练，而不是单进程串行扫完整个架构集合
- 保持可续跑、可聚合、可继续做 meta baseline

## 2. 第一轮正式测试建议

先不要直接恢复完整 `328` 架构长跑。

建议先做两个测试：

### 测试 A：单 shard 校准

先跑一个很小的 shard，确认：

- 新脚本能正常启动
- 输出目录结构正常
- `--skip_existing` 续跑正常

```bash
cd /root/meta-model-train
ARCH_ID_MIN=0 ARCH_ID_MAX=7 FAMILY_NAME=calib_shard_a bash scripts/server_launch_family_shard.sh
```

### 测试 B：双 shard 并行

如果单 shard 正常，再开两个 `tmux` 会话分别跑两半架构。

第一个会话：

```bash
tmux new -s shard0
cd /root/meta-model-train
ARCH_ID_MIN=0 ARCH_ID_MAX=163 FAMILY_NAME=server_v1_full328_a bash scripts/server_launch_family_shard.sh
```

第二个会话：

```bash
tmux new -s shard1
cd /root/meta-model-train
ARCH_ID_MIN=164 ARCH_ID_MAX=327 FAMILY_NAME=server_v1_full328_b bash scripts/server_launch_family_shard.sh
```

退出 `tmux` 不杀任务：

```bash
Ctrl+b 然后按 d
```

查看会话：

```bash
tmux ls
```

重新进入：

```bash
tmux attach -t shard0
tmux attach -t shard1
```

## 3. 跑完后的聚合

```bash
cd /root/meta-model-train
source /root/miniconda3/etc/profile.d/conda.sh
conda activate mpp-meta

python scripts/build_meta_dataset.py \
  --family_dir \
  outputs/toy_diffusion/meta_model_family/server_v1_full328_a \
  outputs/toy_diffusion/meta_model_family/server_v1_full328_b \
  --output_csv outputs/toy_diffusion/meta_model_family/server_v1_full328_meta_dataset.csv
```

## 4. 跑完后优先检查什么

建议优先看这几项：

- 每个 shard 的 `family_summary.csv` 行数
- GPU 利用率是否比单进程更高
- 单位时间完成的 architecture 数是否明显提高
- 聚合后的 `meta_dataset.csv` 是否行数合理

## 5. 当前阶段最重要的判断标准

当前不是优先追求“单步更快”，而是优先追求：

- 相同时间内，能稳定产出更多完整架构曲线

因此更推荐：

- 分片并行
- 稳定续跑
- 跑完后统一聚合

而不是继续围绕单进程 `batch_size` 做很多轮细调。
