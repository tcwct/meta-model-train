# 2.0 setting 说明

这份说明记录当前 `meta-model-train` 仓库里已经启用的 2.0 架构设定，后续本地实验、GitHub 同步和服务器正式训练都以它为准。

## 核心设定

- block 类型只保留两种：`M` 和 `A`
- `M` 表示 residual 2-layer MLP
- `M` 的第二层线性层采用零初始化
- `A` 表示 residual axial attention
- `A` 的 row/col 输出投影采用零初始化，使 residual 分支在初始时接近 0
- 默认层数为 `8`
- 默认 hidden / embedding dimension 为 `8`
- 不再使用旧版 `L / A / R` token，也不再保留相邻 token 的语法约束

## 关键脚本

- `src/meta_model_train/minimal_arch_model.py`
- `scripts/enumerate_architectures.py`
- `scripts/train_minimal_arch_smoke.py`
- `scripts/train_minimal_arch_family.py`
- `scripts/build_meta_dataset.py`
- `scripts/run_meta_baselines.py`

## 当前默认行为

- `python scripts/enumerate_architectures.py` 会默认枚举 8 层架构
- 输出的 architecture CSV 会包含 `num_mlp` 和 `num_attention`
- `family_summary.csv` 和 `meta_dataset.csv` 会沿用动态 `num_*` 字段
- `run_meta_baselines.py` 会自动读取这些 `num_*` 字段，不再写死旧版列名

## 与旧版本的边界

- 旧版 `L / A / R` 设定不再作为当前正式训练入口
- `meta-model-train` 里若还有针对历史结果的分析脚本，应视为历史对比工具，而不是 2.0 主流程的一部分
- 以后若继续调 2.0 setting，优先在这个仓库内演化，不再回到外层 `toy experiments/meta model`
