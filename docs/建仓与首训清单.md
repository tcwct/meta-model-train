# 建仓与首训清单

这份清单专门服务于把 `meta-model-train` 变成以后长期使用的独立仓库。

## A. 本地建一个独立 GitHub 仓库

建议仓库名：

```text
meta-model-train
```

推荐流程：

1. 在 GitHub 上创建一个新的空仓库。
2. 仓库可以先设为 `private`。
3. 不要在 GitHub 网页上额外初始化 README 或 `.gitignore`，避免和本地目录冲突。

## B. 本地最小初始化命令

在 `meta-model-train` 目录中执行：

```powershell
Set-Location "C:\Users\11599\Desktop\本科生科研\毕业论文\multiple_physics_pretraining-main\meta-model-train"
git init
git add .
git commit -m "init meta-model-train project"
git branch -M main
git remote add origin https://github.com/<github-user>/<repo>.git
git push -u origin main
```

说明：

- 这里是在 `meta-model-train` 目录里单独建仓，不会影响外层大仓库的工作流。
- `outputs/`、`artifacts/`、`__pycache__/` 等内容已被 `.gitignore` 处理。

## C. 服务器首次拉取

服务器侧仍然推荐走已经验证过的 `SSH over 443`：

```bash
cd /root
git clone git@github.com:<github-user>/<repo>.git
cd <repo>
git log --oneline -1
```

## D. 服务器首个最小验证

进入仓库后：

```bash
chmod +x scripts/server_quickstart.sh
bash scripts/server_quickstart.sh
```

这一步会自动做两件事：

- 检查 conda / torch / CUDA / GPU 是否可用
- 跑一个最小 smoke pipeline

## E. 成功标志

如果一切正常，你会看到：

- `check_environment.py` 输出环境 JSON
- `smoke pipeline` 成功结束
- 在 `outputs/smoke_pipeline/server_quickstart/` 下出现：
  - `pipeline_manifest.json`
  - `smoke/`
  - `family_runs/family/meta_dataset.csv`

## F. 后续日常工作

以后推荐优先使用这些命令：

```bash
python scripts/meta_model_cli.py check-env
python scripts/meta_model_cli.py smoke-pipeline --device cuda
python scripts/train_minimal_arch_smoke.py --device cuda --max_steps 50
```
